===============================================================================
1. PRÉSENTATION DU PROJET
===============================================================================

Sahel.ai est une plateforme SaaS (Software as a Service) destinée aux petites
et moyennes entreprises (PME) marocaines. Elle permet de créer facilement un
chatbot IA intelligent et un mini-site web vitrine à partir de documents
existants (PDF, catalogues, menus, inventaires).

Le nom "Sahel" signifie "facile" en arabe, reflétant la mission du projet :
rendre l'intelligence artificielle accessible aux commerçants marocains sans
compétences techniques.

Le projet exploit une architecture :
  - Frontend : Application React avec dashboard propriétaire et site vitrine
  - Backend : API FastAPI avec pipeline RAG (Recherche sémantique + IA
    générative)
  - Bots : Agents conversationnels WhatsApp et Telegram
  - Base de données : PostgreSQL hébergée sur Supabase (cloud gratuit)


===============================================================================
2. OBJECTIFS
===============================================================================

2.1 Objectif Général

  Développer une plateforme web permettant aux PME marocaines de créer un
  assistant IA personnalisé, alimenté par leurs propres documents, et
  accessible via web, WhatsApp et Telegram.

2.2 Objectifs Spécifiques

  a) Créer un pipeline RAG (Retrieval-Augmented Generation) pour transformer
     des documents PDF en connaissances exploitables par une IA.
  b) Développer un dashboard analytique avec des indicateurs de performance en
     temps réel (conversations, langues, questions fréquentes).
  c) Concevoir un générateur de mini-sites web automatique à partir des
     informations du commerce.
  d) Intégrer des chatbots WhatsApp et Telegram capables de répondre en
     Darija, Français et Anglais.
  e) Assurer un déploiement à coût zéro (HuggingFace Spaces + Vercel +
     Supabase).

                           │N
                      ┌────┴───────┐
                      │  MESSAGE   │
                      └────────────┘

  ┌──────────────┐
  │   INQUIRY    │   (Entité indépendante — demandes de contact)
  └──────────────┘

  Règles de gestion :
    - Un propriétaire peut gérer plusieurs commerces
    - Un commerce peut avoir plusieurs documents
    - Une conversation appartient à un seul commerce
    - Une conversation contient plusieurs messages
    - Une inquiry est indépendante (pas liée à un compte)

5. ARCHITECTURE RAG (Retrieval-Augmented Generation)
===============================================================================

Le pipeline RAG est le coeur intelligent du projet. Il permet au chatbot de
répondre aux clients avec des informations précises issues des documents du
commerce.

  Fonctionnement :

  ┌────────────────┐
  │   Document     │  PDF, Word, CSV uploadé par le commerçant
  │   (PDF/Word)   │
  └───────┬────────┘
          │
          ▼
  ┌────────────────┐
  │   Extraction   │  PyPDF lit le texte du document
  │     (PyPDF)    │
  └───────┬────────┘
          │
          ▼
  ┌────────────────┐
  │   Découpage    │  CharacterTextSplitter (LangChain) découpe en
  │   (Chunks)     │  segments de ~500 caractères avec chevauchement
  └───────┬────────┘
          │
          ▼
  ┌────────────────┐
  │   Embedding    │  Chaque chunk est converti en vecteur numérique
  │   (FAISS + HF) │  de 384 dimensions (all-MiniLM-L6-v2)
  └───────┬────────┘
          │
          ▼
  ┌────────────────┐
  │   Index FAISS  │  Les vecteurs sont stockés dans un index de
  │                │  recherche par similarité cosinus
  └───────┬────────┘
          │
     ┌────┴────┐
     │         │
     ▼         ▼
  ┌────────────────┐    ┌────────────────┐
  │ Question       │    │ Top-K chunks   │
  │ du client      │───▶│ les plus       │───▶ Concaténation
  │ (encodée)      │    │ pertinents     │    dans le prompt
  └────────────────┘    └────────────────┘
                              │
                              ▼
                    ┌────────────────┐
                    │   Groq API     │
                    │  (Llama 3      │───▶ Réponse finale
                    │   70B)         │
                    └────────────────┘

  Avantages du RAG par rapport à un LLM standard :
    - Réponses basées sur les documents réels du commerce
    - Pas besoin de ré-entraîner le modèle
    - Mise à jour instantanée des connaissances (nouveau PDF = nouveau savoir)
    - Confidentialité : les données ne servent pas à entraîner des modèles
      publics

7. KPIs — INDICATEURS CLÉS DE PERFORMANCE
===============================================================================

  Indicateur              Source                  Calcul
  ─────────────────────────────────────────────────────────────────────────
  Conversations totales   analytics.total_         Somme brute des
                          conversations           conversations

  Messages total          analytics.total_         Somme brute des messages
                          messages

  Taux de réponse         messages avec           (messages répondus /

9. CHATBOTS WHATSAPP & TELEGRAM
===============================================================================

9.1 WhatsApp Bot

  Technologie : Baileys (@whiskeysockets/baileys) — bibliothèque WhatsApp
  Web Multi-Device en Node.js.

  Fonctionnement :
    - Connexion via scan d'un QR code depuis WhatsApp (Menu → Appareils
      liés → Lier un appareil)
    - Session persistante (dossier session/) — pas besoin de rescanner
      à chaque démarrage
    - Reconnexion automatique en cas de perte de connexion
    - Gestion multi-utilisateurs (chaque client a sa session)

  Commandes :
    /start ou /menu    → Afficher le menu principal
    /business <slug>   → Changer de commerce actif

  Logique métier :
    - Le texte du client est extrait du message WhatsApp
    - Appel à l'API REST /api/businesses/{slug}/chat avec la question
    - L'API interroge le pipeline RAG + Groq
    - La réponse est renvoyée au client sur WhatsApp
    - Limitation de débit : 30 requêtes/minute max, 5 req/10s

  Données persistées :
    - users.json : sauvegarde des préférences de chaque utilisateur
      (quel commerce il consulte)


9.2 Telegram Bot

  Technologie : Long-polling direct de l'API Telegram (sans bibliothèque
  externe — appels HTTP à api.telegram.org).

  Fonctionnement :
    - Boucle de polling toutes les 1.5 secondes
    - Récupération des nouvelles mises à jour via getUpdates
    - Traitement des commandes et messages

  Commandes :
    /start              → Message de bienvenue + menu
    /menu               → Afficher le menu principal
    /business <slug>    → Changer de commerce actif

  Différences avec WhatsApp Bot :
    - Pas de scan QR (bot lié à un token)
    - Interface textuelle simple
    - Même backend API (mêmes réponses IA)
